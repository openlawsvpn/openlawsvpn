# Security Policy

## Reporting a Vulnerability

The `openlawsvpn` team takes the security of our software seriously. If you believe you have found a security vulnerability, please report it to us responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please send an email to the maintainers:
- **Anatolii Vorona** <security@openlawsvpn.com>

Please include as much information as possible, including:
- A description of the vulnerability.
- Steps to reproduce the issue (PoC).
- Potential impact.

## Scope

This security policy applies to the following repositories:
- `openlawsvpn/openlawsvpn` (the community client)

## Response Process

1. We will acknowledge receipt of your report within 48 hours.
2. We will investigate the issue and provide an estimated timeline for a fix.
3. We will notify you once the vulnerability is resolved.
4. We will coordinate a public disclosure if appropriate.

## Preferred Languages

We prefer reports in English. This allows us to process your report more efficiently and accurately.
